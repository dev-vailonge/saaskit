Documentação MVP Espresso

Bem-vindo à documentação do MVP Espresso! Este guia irá te ajudar a configurar e personalizar seu projeto do zero. Siga cada passo cuidadosamente para garantir uma configuração adequada do ambiente.

Pré-requisitos

  • Conhecimento básico de terminal/linha de comando
  • Editor de código (recomendamos o Cursor)
  • Conexão estável com a internet

Criar Conta GitHub

O GitHub é onde você irá armazenar seu código e fazer o deploy do projeto.

  1. 1. Acesse github.com
  2. 2. Clique em "Sign up" no canto superior direito
  3. 3. Preencha suas informações e siga o processo de verificação
  4. 4. Confirme seu email

Instalar Git

Git é essencial para controle de versão e deploy do projeto.

  1. 1. Acesse git-scm.com/downloads
  2. 2. Clique no botão de download para seu sistema operacional
  3. 3. Execute o instalador baixado
  4. 4. Mantenha as opções padrão durante a instalação
  5. 5. Após a instalação, abra o terminal e verifique se o Git foi instalado corretamente:
    6. git --version

Dica

Durante a instalação, mantenha as configurações padrão sugeridas pelo instalador. Isso garantirá uma instalação adequada para a maioria dos usuários.

Instalar Node.js

Node.js é necessário para executar o projeto localmente.

  1. 1. Acesse nodejs.org
  2. 2. Baixe a versão LTS (recomendada para a maioria dos usuários)
  3. 3. Execute o instalador
  4. 4. Verifique a instalação no terminal:
    5. node --version

Download do Template

Após a compra, você receberá acesso ao repositório do template.

  1. 1. Acesse o link do Google Drive fornecido
  2. 2. Faça o download do arquivo zip
  3. 3. Descompacte o arquivo
  4. 4. Entre na pasta do projeto:
    5. cd mvp-espresso

Rodando o Projeto

Siga estes passos para executar o projeto localmente.

  1. 1. Instale as dependências:
    2. npm install
  2. 2. Configure as variáveis de ambiente (próxima seção)
  3. 3. Execute o projeto em modo de desenvolvimento:
    4. npm run dev
  4. 4. Acesse http://localhost:3000

Configurar Variáveis ENV

Configure as variáveis de ambiente necessárias.

  1. 1. Copie o arquivo .env.example:
    2. cp .env.example .env.local
  2. 2. Preencha as variáveis com suas credenciais:
    3. NEXT_PUBLIC_SUPABASE_URL=sua_url_supabase
NEXT_PUBLIC_SUPABASE_ANON_KEY=sua_chave_supabase
STRIPE_SECRET_KEY=sua_chave_secreta_stripe
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=sua_chave_publica_stripe
UPSTASH_REDIS_REST_URL=sua_url_upstash
UPSTASH_REDIS_REST_TOKEN=seu_token_upstash

Deploy do Projeto

Deploy seu projeto na Vercel.

  1. 1. Faça push do seu código para o GitHub
  2. 2. Acesse vercel.com
  3. 3. Clique em "New Project"
  4. 4. Importe seu repositório
  5. 5. Configure as variáveis de ambiente
  6. 6. Clique em "Deploy"

Solução de Problemas

Problemas Comuns

Erro de instalação de dependências

Tente remover node_modules e package-lock.json:

rm -rf node_modules package-lock.json && npm install
Erro de variáveis de ambiente

Verifique se todas as variáveis em .env.local estão preenchidas corretamente

Erro de build na Vercel

Verifique se todas as variáveis de ambiente estão configuradas no projeto da Vercel

Para mais ajuda, acesse nossa comunidade no Discord ou abra uma issue no GitHub.
